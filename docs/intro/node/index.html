Understanding Nodes in ValQ

With ValQ, you can build a model of your business process and view it in the form of a tree as well as a table. Toggle between the below icons present on the top left corner of the canvas to change the view.

#Tree View

The tree structure is presented in a horizontal format as a visual. Just like a tree, ValQ model has a stem on Top called Top node and the Top node then branches out to other nodes.


The model is segmented into levels starting from level 1 with Top node, here it's 'Overall Result'. The nodes share a parent-child relationship at each level. For example, the 'East' node is the Parent node while 'Consumer' node is a Child node, 'Consumer' node is the Parent node while 'Chris Morris' node is a Child node.


Click on '>' to expand and '<' collapse levels.


You can also use the 'Go to level ^' option present on the bottom of the canvas to navigate between levels of the tree.

#Table View

The same information can be viewed in tabular form as see in the image below. The Table view is generally used to plan and analyze the data. Click on the table icon as highlighted below to get the tabular view.


For more information on Table View, click here

#Node Components

Now let's take a look at what makes up a node.


Each node represents a value driver, KPI or category and it's several performance metrics. Here's how to interpret a node:

ComponentDescription
1Node nameName of the value driver or KPI
2Conditional FormattingColor band indicating the variance
3Value TrendA sparkline graph showing the trend
4Baseline ValueNode value of the baseline series
5Variance for all periodsFull year variance of the node
6Variance for current periodCurrent month (usually first period in the series) month variance of the node
7Absolute VarianceVariance showed in absolute value
8Percentage VarianceVariance showed in percentage

Note
Variance is calculated as the difference between the Baseline series and Comparison series

Baseline series is the primary series and Comparison series is the benchmark against which the Baseline is compared. Here the Sales Budget is the Baseline series and Sales Forecast is the Comparison series.
