## What can I use ValQ for?

<div style={{textAlign: 'justify'}}>
ValQ is a powerful, multi-purpose tool that can be used across all functions in an organization. It is designed to 
help decision-makers and executives simulate various scenarios on-the-fly and analyze outcomes to
arrive at actionable decisions. ValQ supports use cases such as what-if simulations, scenario comparisons, planning, forecasting, variance analysis, value driver analysis and more.
</div> <br />

## Who can use ValQ?

ValQ is for everyone ranging from C-suite executives to analysts across departments and functions. 

> ** Note** <br /> ValQ is not limited to financial & operations planning functions. 

<div style={{textAlign: 'justify'}}>
For example, <br />
1.  A customer service rep can leverage ValQ to simulate headcount needs based on estimated growth in call volume,
upcoming product launches, marketing campaigns, and more. <br />
2.  A marketing manager can leverage ValQ to analyze web traffic, online assets, conversion rates, etc. to calculate
marketing ROI.
</div> <br />

## What do I need to run ValQ?

ValQ is an independent solution that does not require any infrastructure, database, service or IT support.

> ** Note** <br /> All you need is Microsoft Power BI to run ValQ

<div style={{textAlign: 'justify'}}>
Note that the Microsoft Power BI desktop software is free, and you can download it from <a href="https://powerbi.microsoft.com/en-us/downloads/" target="_blank">here</a>.
With ValQ available as a standalone visual for Microsoft Power BI in the app store, you can get running in a matter of minutes.
Learn how to download & install ValQ <Link to={useBaseUrl('docs/intro/install')}>here</Link>.
</div> <br />

## What sources can ValQ connect to?

<div style={{textAlign: 'justify'}}>
ValQ can connect to all the databases, platforms and other data sources that are supported by Power BI – including but not limited to:
</div> <br />

  - Databases such as SQL Server, SSAS, Oracle, Snowflake
  - Platforms and apps such as Salesforce, Dynamics 365, SAP HANA, and SAP BW
  - and other sources such as Excel, Web, JSON, OData
