## 2. Configuration

<div style={{textAlign: 'justify'}}>
In Configuration, you will be able to configure the node
details such as Node Type, Calculation Method, Time Aggregation, Desired Trend and
Value Range with the values as shown in the below Figure.
</div> <br />

<div style={{textAlign: 'center'}}>
  <Zoom>
<img alt="New Tab" src={useBaseUrl('/doc-images/model/node11.png')} />
  </Zoom>

<div style={{textAlign: 'justify'}}>

**Node Type**: Node can be of two types - Normal and Constant. Normal node is the default type
with all the options available while constant node sets the node value to a fixed figure. 
</div> <br />

<div style={{textAlign: 'justify'}}>

**Calculation Method**: This option is available to choose a method of calculation for the node from the below-seen list of methods.
</div> <br />

<div style={{textAlign: 'justify'}} className="table-center">

|   	| Method  	| Description  	|
|---	|---	| ------- |
|  1    |  Data Source | Gets the value directly from source data       |
|  2 	|  Sum Children 	| Adds all of the child node values   	|
|  3 	|  Subtract Children 	| Deducts the other child node values from the first child node |
|  4 	|  Multiply Children 	| Multiplies all of the child node values | 
|  5 	|  Divide Children 	| Divides the other child node values from the first child node |
|  6    |  Formula          | Calculates and applies the custom formula entered by the user    |
|  7    | Linked to Node | The value is linked to value from another node |
|  8    | Template based on other node | Captures and applies the configuration from the other node |
|  9    | Manual | Enter the values manually |
</div> <br />

> **Note** <br />
<div style={{textAlign: 'justify'}}>
On selecting 'Formula' and clicking on the formula field, you will be
able to see a dropdown of the formulas that are supported. For a list of
formulas that are available in ValQ, you can refer to the link <Link to={useBaseUrl('/docs/faq/formula-list')}> Functions </Link>
</div> <br />

<div style={{textAlign: 'justify'}}>
Using the Conditional Formatting option, you will be able to apply the
Alert Thresholds and Rules at the specific Node Level (see Figure
below). For our example, the Node 'East' has been selected.
</ div> <br />

<div style={{textAlign: 'justify'}}>

**Global or Inherited Rule**: When the option "Global or Inherited Rule" is selected, the
Conditional Formatting Rules configured as Global Rule in the Settings
tab will be applied to the node.
</div> <br />

<div style={{textAlign: 'justify'}}>

**Custom Rule**: Custom Rule option overwrites default or any inherited
rule to the rule as per our preference. The Custom Rule option can be configured based on the 
variance percentage or value and it gets applied at the period level. Checking Apply to Descendants
option applies the Custom Rule to the child nodes as well. 
</div> <br />

Custom Rule is indicated by the icon (see Figure below) on the node in the Navigation Panel.
<div style={{textAlign: 'center'}}>
  <Zoom>

<div style={{textAlign: 'justify'}}>

**Hidden Rule**: With this option, you can observe that no range values can be provided for the Conditional
Formatting and hence there will be no status color for the node upon being simulated.
</div> <br />

<div style={{textAlign: 'justify'}}>

In the below example, we choose the option for the Custom Rule as
"Percentage" and also enable the property "Apply to Descendants". We set
the threshold values as shown in the below Figure. After the simulation, you
can find that the variance percentage value for the node "East" falls
in the threshold range (5%)\* which is between -10% to -1% and based on
that condition, the status bar for the Node East is yellow color (see
